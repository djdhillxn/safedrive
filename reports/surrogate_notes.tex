This is a meaningful improvement over the earlier 2\% PPO result, but road departure

SAC did produce complete artifacts, but the policy collapsed. Its best old evaluation had

Three measurement defects were found. First, MetaDrive's \texttt{info["velocity"]} is
already in km/h, but the evaluator multiplied it by 3.6; every legacy mean-speed result is

validation set rather than an untouched test set.

Priority 0 corrects these issues as follows:
\begin{itemize}
  \item training uses seeds beginning at 0 and keeps randomized traffic;
  \item deterministic validation uses seeds 1000--1049;
  \item deterministic held-out testing uses seeds 2000--2099 and runs only after the
        validation winner is frozen;
  \item the best checkpoint is ranked by success, then route completion, lower off-road,
        collision and timeout rates, and finally mean return;
  \item checkpoint normalization statistics are saved every 50,000 steps and beside the
        best model;
  \item training outcomes and PPO/SAC optimizer diagnostics are saved to
        \texttt{training\_diagnostics.json};
  \item success summaries include a 95\% Wilson interval;
  \item video recording explicitly sets the configured scenario seed immediately before
        reset and writes the seed and final outcome beside the MP4.
\end{itemize}

\section{2026-07-21 newest-run audit and ground-up correction}
The later Google Drive import supersedes the artifacts listed above for diagnosis. The
newest completed learned runs are
\texttt{20260721\_090542\_ppo\_mvp\_ppo\_seed0} and
\texttt{20260721\_105717\_sac\_mvp\_sac\_seed0}; the newest IDM test is
\texttt{20260721\_181208\_idm\_baseline\_seed0}. Only these timestamps were used for the
second audit.

The frozen PPO winner reached 4\% success, 10\% collision, 75\% off-road, 11\%
timeout, and 32.5\% route completion on 100 test episodes. Its validation series peaked
at 10\% success at 350,000 steps and fell back to 0\% at 500,000. PPO's explained
variance and other optimizer diagnostics were mostly healthy. The agent was learning the
configured return, but that return was not a good enough proxy for safe completion. In
particular, successful episodes returned about 480 on average, while out-of-road, crash,
and timeout episodes still returned about 88, 103, and 240. Failure remained profitable.

The frozen SAC winner reached 0\% success, 7\% off-road, 93\% timeout, 15.5\% route
completion, and only 0.99 km/h mean speed. The failure is not missing evaluation output.
The critic loss grew from 0.17 at 210,000 steps to 61.9 at 310,000, 1,858 at 410,000,
and 5,817 at 500,000, with a peak above 12,000. At the same time the learned entropy
coefficient grew from roughly 0.01 to 1.36 and the actor-loss magnitude exceeded 1,000.
The best validation point was at 200,000 steps; later learning collapsed toward braking
and stalling. Timeout episodes still returned about 72 on average, so the replay learner
had both an attractive behavioral loophole and unstable value targets.

The ground-up code audit found the following causal defects:
\begin{enumerate}
  \item MetaDrive replaces only the terminal step reward on success or failure; it does
        not erase the dense progress accumulated earlier. Terminal penalties of 20 could
        not dominate hundreds of units of dense return.
  \item \texttt{use\_lateral\_reward} only scales positive progress near a lane edge; it
        is not a negative lane-centering cost. The prior documentation overstated it.
  \item No timeout penalty was configured, and \texttt{truncate\_as\_terminate} was
        false. Stable-Baselines3 therefore bootstrapped through horizon truncations. That
        is inappropriate here because this project explicitly counts timeout as failure,
        and it was especially damaging to SAC's replay targets.
  \item MetaDrive's state/lidar observation is already bounded in [0,1]. A second adaptive
        \texttt{VecNormalize} transform added nonstationarity without evidence of benefit.
  \item The old task exposed only 50 training scenarios on a five-block map with density
        0.1. It was simultaneously narrow for generalization and difficult enough that
        even native reference policies often failed.
  \item Earlier telemetry retained terminal actions but not full-rollout action behavior or
        the split between base reward and shaping penalties, making collapse harder to see.
\end{enumerate}

Native-policy sanity checks on validation seeds 1000--1019 separated simulator difficulty
from optimizer behavior:
\begin{center}
\begin{tabular}{lrr}
  \toprule
  Task & Expert success & IDM success \\
  \midrule
  Map 5, density 0.10 & 55\% & 35\% \\
  Map 3, density 0.00 & 100\% & 100\% \\
  Map 3, density 0.05 & 100\% & 100\% \\
  Map 4, density 0.05 & 95\% & 95\% \\
  Map 4, density 0.10 & 65\% & 50\% \\
  \bottomrule
\end{tabular}
\end{center}
This evidence justifies using map 3 at density 0.05 as the Phase-1 learning benchmark.
This is an intentional simplification, not an algorithmic breakthrough. Higher map and
traffic difficulty belongs in a later robustness phase after basic learning is established.

The new implementation uses 500 training scenarios, fixed spawn lanes, direct bounded
observations, terminal treatment of timeouts, success reward 100, failure penalties
50, and forward-progress weight 0.2. A wrapper penalizes speed below 5 km/h, lateral lane
deviation, horizon timeout, steering magnitude, and abrupt steering changes while logging the base reward and
each penalty. PPO now uses learning rate $10^{-4}$, 4,096-sample rollouts, batch size 256,
and CPU. SAC uses learning rate $10^{-4}$, a 500,000-step replay buffer, 20,000 learning
starts, batch size 512, GPU when available, and fixed entropy coefficient 0.05. Full action
distribution metrics are saved during training and evaluation.

Because seeds 2000--2099 informed this diagnosis, the untouched final test range moves to
3000--3099. Before another 500,000-step run, each algorithm must pass a 100,000-step
no-traffic control pilot on fixed validation seeds with at least 10\% success and 50\%
route completion. The SAC pilot also fails on a non-finite or final critic loss above
1,000. Pilot pointers are named \texttt{latest\_ppo\_pilot.txt} and
\texttt{latest\_sac\_pilot.txt}, so they cannot replace full-run pointers. If a gate
fails, stop; more timesteps are not the next experiment.

\section{2026-07-21 unbiased reset and bounded Phase-1 scope}
The newest PPO control pilot,
\texttt{20260721\_230755\_ppo\_control\_pilot\_ppo\_seed0}, invalidated the preceding
``Priority-0'' prescription. At 25,000 steps it had 0\% success, 100\% off-road, and
3.6\% route completion. From 50,000 through 100,000 steps its deterministic policy mostly
braked, timed out in every validation episode, and completed only 1.4\% of the route.
Training was numerically finite, but sampled actions had mean absolute steering near 0.8
and steering saturation around 35\%. This was learned failure, not an optimizer crash.

The causal reset found four problems:
\begin{enumerate}
  \item The experiment asked one 100,000-step run to learn basic control, continuous
        steering, 500-map generalization, traffic interaction, and a new safety reward at
        once. It was not a controlled proof of learning.
  \item With $\gamma=0.99$, a success reward of 100 delivered near step 474 contributes
        only about 0.86 to the first decision. A timeout penalty of 50 at step 1000
        contributes about 0.002. Dense progress before road departure could still have
        positive discounted return, while braking avoided immediate risk.
  \item Stable-Baselines3 initializes a continuous PPO policy with a unit standard
        deviation. Clipping those samples into MetaDrive's $[-1,1]$ steering range caused
        destructive exploration before a useful trajectory existed.
  \item The environment factory confused the algorithm RNG seed with MetaDrive's scenario
        index. The broad 0--499 range hid this. A one-map reproduction failed because PPO
        reseeded workers with 0--3 while the only legal scenario was 5. The environment
        boundary now maps generic RNG seeds into the configured scenario range while
        preserving explicit validation/test scenario IDs.
\end{enumerate}

Controlled local experiments then separated pipeline competence from task difficulty.
On MetaDrive's fixed curved map \texttt{C}, reference reward, no traffic, a 500-step
horizon, and PPO initial \texttt{log\_std=-2}, PPO reached 39.4\% route completion at its
best 100,000-step checkpoint but 0\% success. Extending a documented-style budget to
300,000 steps still produced 0\% success. The same continuous learner on fixed straight
map \texttt{S} reached 100\% success and 96.7\% route completion at 75,000 steps. Its
frozen checkpoint then achieved 100\% success, 95.8\% mean route completion, and zero
collision, off-road, or timeout outcomes across seeds 1000--1009. Thus the code path can
learn, while curve following remains an unproven next capability. A further diagnostic on
seeds 3000--3019 produced 90\% success, 95.9\% route completion, 0\% collision, 10\%
off-road, and 0\% timeout. This passes the proposed numerical POC boundary, but consuming
those results means that range is no longer held out; final reporting moves to 4000--4019.

The first SAC straight-road control used automatic entropy tuning. Its coefficient fell
from 0.223 at 10,000 steps to 0.011 at 20,000 and approximately
$3\times10^{-5}$ by 45,000--50,000 steps. The critic remained finite, but actions became
strongly asymmetric and saturated; deterministic validation had 0\% success and 100\%
off-road at 50,000 steps. Fixed entropy 0.05 improved the 25,000-step checkpoint to
57.1\% route completion, but it still had 0\% success and regressed by 50,000 steps.
Enabling only MetaDrive's built-in lateral multiplier produced 33.1\% route completion
and 0\% success at 25,000 steps. Generalized state-dependent exploration with initial
\texttt{log\_std=-2} still sampled mean absolute steering near 0.49 by 10,000 steps.
These one-change controls show that neither entropy, lateral reward, nor gSDE alone fixed
the replay data.

The decisive SAC boundary was explicit and task-level: its straight-road POC exposes
steering only in $[-0.1,0.1]$ while leaving throttle/brake in $[-1,1]$. This represents
small lane-centering corrections. SAC then reached 100\% validation
success and 95.8\% route completion at 25,000 steps, so the new success-first callback
saved the checkpoint and stopped training. On a separate diagnostic range, seeds
3500--3519, the frozen checkpoint achieved 100\% success, 95.7\% route completion, and
zero collision, off-road, and timeout outcomes. Final seeds 4000--4019 remain untouched.
The same bound did not transfer cleanly to PPO: the best bounded PPO control reached 70\%
success, 92.4\% route completion, and 40\% off-road at 75,000 steps, while changes to the
bound, initial standard deviation, and lateral reward did not qualify. PPO with full
steering and initial \texttt{log\_std=-2} reached 80\% validation success and 95.3\%
route completion at 50,000 steps, but its 20\% off-road rate exposed a bug in the first
early-stop rule: only success and route were enforced. The callback and notebook gate now
also enforce the 10\% collision, off-road, and timeout limits, so that checkpoint can no longer pass
prematurely. A separate full-steering diagnostic produced 85\% success, 95.5\% route
completion, and 15\% off-road. The continuous-control probe therefore did not qualify
within its 100,000-step budget.

A second infrastructure bug explained why otherwise identical PPO runs diverged after the
first 25,000-step validation. The vector evaluator called the project-wide random-seed
helper before every validation episode. Because callback evaluation runs in a simulator
subprocess but policy optimization resumes in the parent process, this silently reset
Python, NumPy, and PyTorch's training random streams to the last validation scenario seed.
Changing the number of validation scenarios therefore changed subsequent PPO minibatches
and exploration. Vector evaluation now seeds only its simulator subprocess, and a regression
test verifies that the parent random streams are unchanged.

After isolating those random streams, the exact full-steering PPO control peaked at 40\%
success, 87.7\% route completion, and 60\% off-road within 100,000 steps. Phase 1 therefore
adopts MetaDrive's documented beginner PPO abstraction rather than presenting a fragile
continuous result as solved: a $3\times3$ multidiscrete steering/throttle grid. With the
same straight road, reference reward, strict validation split, and 100,000-step maximum,
this PPO control reached 100\% success, 96.4\% route completion, and zero collision,
off-road, or timeout outcomes at its first 25,000-step validation and stopped. Its frozen
checkpoint repeated 100\% success, 96.6\% route completion, and zero failure outcomes on
20 diagnostic seeds 3600--3619. Final seeds 4000--4019 remain untouched.

This evidence supersedes the previous plan to call a three-block, 500-scenario traffic
task the Phase-1 baseline. The required Phase-1 claim is now bounded lane
following on one straight, traffic-free road. PPO and SAC each receive at most 100,000
steps; a checkpoint must reach 80\% validation success and 90\% route completion. The
callback saves the checkpoint before stopping only when validation collision, off-road,
and timeout rates are also no greater than 10\%. Final test uses the same criteria on 20
untouched seeds. PPO uses a $3\times3$ discrete control grid and SAC uses a continuous
$[-0.1,0.1]$ steering control. These documented task interfaces prove pipeline competence
but do not support a direct algorithm ranking. This is a student-scale proof of learning,
not procedural generalization or safe autonomy. The
difficulty ladder is continuous PPO and unrestricted SAC steering on the same road, a fixed
curve, a small traffic-free procedural set, then light traffic.
Only one dimension advances after both learned agents pass the current one.

\section{Phase-2 implementation record}
Phase 2 asks one question: does staged road-geometry curriculum improve unseen
procedural-road generalization over equal-budget direct SAC training? It does not add a
third learned algorithm. The two conditions share full continuous control, the same
$[256,256]$ MLP, reference reward, 500,000-step cap, no training traffic, 100 training
scenarios beginning at seed 10000 (cycled sequentially), 25 validation scenarios beginning at seed 20000, and
100 untouched test scenarios beginning at seed 30000.

The first Phase-2 feasibility attempt exposed a version boundary before any rollout was
created: MetaDrive 0.4.3 rejected \texttt{sequential\_seed} as a constructor option. The
setting is now consumed by SafeDrive's scenario wrapper instead of being forwarded to
MetaDrive. Resets without an explicit seed cycle deterministically through the configured
training range, while validation and testing continue to request each exact scenario seed.

The direct condition trains on random three-block maps throughout. The curriculum condition
uses a fixed curve \texttt{C} for at most 100,000 steps, \texttt{SC} for at most 150,000
steps, and spends the remaining budget on random three-block maps. The first two stages
must reach 90\% success and route completion with no more than 10\% collision, off-road,

Actor, critic, optimizer, and replay-buffer state persist across stages. A stage-boundary

The experiment order is:
\begin{enumerate}
  \item close Phase 1 with exact-test IDM, fingerprint-checked comparison, and videos;
  \item run Phase-2 IDM and ExpertPolicy validation feasibility checks;
  \item run direct and curriculum SAC with algorithm seed 0;
  \item test both frozen policies on 100 untouched procedural scenarios;
  \item run seeds 1 and 2 only if either pilot reaches 50\% success or 80\% route completion;
  \item evaluate frozen policies without retraining at traffic density 0.05;
  \item compare all available strictly compatible training seeds and build the report.
\end{enumerate}

Traffic training, SafeMetaDrive obstacles, vehicle-dynamics randomization, image policies,
real-world replay, and multi-agent experiments remain deferred. If Phase 2 succeeds, the
next coherent extension is explicit obstacle/cost learning in SafeMetaDrive rather than
another broad algorithm sweep.
